### Why it's not loading

**Root cause:** Nansen `smart-money/netflow` endpoint requires **x402 micropayment protocol**.

- `apiKey` header → 401 "Invalid API key" (any key we have)
- `x-api-key` or `Authorization: Bearer` → 402 "Payment Required via x402 protocol"
- x402 = cryptographic payment signature per request — NOT a standard REST API key flow
- Free plan = 0 credits, but even with credits this endpoint requires x402

**The miss:** During testing we saw `{"plan":"free","credits_remaining":0}`. Should have confirmed "does a paid Nansen account unlock this via standard key auth?" before building. Didn't verify. That's on Rex.
